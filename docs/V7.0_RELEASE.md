## Scoring details (v7.0)

### 12-point system with capped RS bonus

| Pillar | Points | Logic |
|--------|--------|-------|
| **P1** HTF Trend | 3 | 4H EMA50 vs EMA200 |
| **P2** Momentum | 2 | MACD AND RSI(>50, rising) → 2 / either → 1 / neither → 0 |
| **P3** Volume | 3 | high vol AND CVD aligned → 3 / either → 1 / neither → 0 |
| **P4** Regime | 2 | TRENDING or VOLATILE |
| **P5** Candle Body | 2 | body > 50% of bar range, directional |
| **RS Bonus** | +1 | +1 if outperforming SPY 5d (long) or underperforming (short) |
| **Final** | **min(12, sum)** | Bonus rolls into existing grade boundaries |

**Grade boundaries unchanged:** A+ ≥ 12, A ≥ 9, B ≥ 6, C < 6

## SL and trail schema (v7.0)

### Initial SL (entry)
- Structure-based: recent swing low/high ± `structBuffer × ATR`
- Floor: 0.8 × ATR (`minSLDistance`)
- Cap: 5.0 × ATR standard, `maxSLDistVol` (default 5.0) on VOLATILE regime
- Override if dynamic regime R:R enabled

### Stage-tighten (v7.0 I3 NEW)
- Triggered when profit ≥ +0.5R
- New SL: `entry ± min(initial_max, 3.0 × ATR)` direction-correct
- `slStageTightened` flag prevents repeat
- Resets on new entry

### Early breakeven (v6.10 unchanged)
- Triggered when profit ≥ +0.75R
- Trail floor moves to `entry ± 0.3R` (small profit lock)
- `earlyBEArmed` flag persists until new entry

### TP stage floors (v6.10 unchanged)
- TP1 hit → floor = `entry ± 0.5R` (breakeven+)
- TP2 hit → floor = `TP1 price`
- TP3 hit → floor = `TP2 price`

### Stage-aware trail multiplier (v7.0 I4 NEW)

| Stage | ATR Mult | Effect |
|-------|----------|--------|
| Pre-TP1 (after early BE) | 1.8 (base) | Wide enough to avoid noise stops |
| After TP1 | 1.5 | Moderate tightening |
| After TP2 | 1.0 | Tight — protect realized gains |
| After TP3 | 0.6 | Lock most TP3 gains hard |

`trailEngaged`: 1H+ requires TP1 hit (TRENDING) or earlyBE; 15m/30m engage on earlyBE.

## SQS time-of-day multiplier (v7.0 I7 NEW)

| ET Window | Multiplier | Rationale |
|-----------|-----------|-----------|
| 10:00-11:00 | 1.05 | Morning trend window (post-open settle) |
| 14:00-15:00 | 1.05 | Afternoon trend window (pre-close direction) |
| 11:30-13:30 | 0.90 | Lunch chop penalty |
| Other | 1.00 | Neutral |

Multiplier applies to **raw SQS before threshold check**. Cap at 100.

A B-grade signal during lunch with raw SQS = 78:
- v6.10: passes (78 ≥ 75)
- v7.0: rejected (78 × 0.90 = 70.2, below 75)

A B-grade signal at 10:30 with raw SQS = 72:
- v6.10: rejected (72 < 75)
- v7.0: passes (72 × 1.05 = 75.6, above 75)

## RS vs SPY confluence (v7.0 I8 NEW)

### Calculation
```
spyClose_today     = SPY daily close prior bar
spyClose_5sessions = SPY daily close 6 bars back
spyRet5            = (today - 5sessions_back) / 5sessions_back × 100

bars_per_session   = max(1, 390 / TF_minutes)        // intraday-aware
lookback_bars      = min(2000, 5 × bars_per_session)
symRet5            = (close - close[lookback]) / close[lookback] × 100

rs5d               = symRet5 - spyRet5
```

### Apply
- **Long signal:** `rs5d > 0` → +1pt to `bullScore`
- **Short signal:** `rs5d < 0` → +1pt to `bearScore`
- Final score: `min(12, raw + bonus)` — caps preserve grade boundaries

### Examples
| Symbol vs SPY | Long signal | Short signal |
|---------------|-------------|--------------|
| Sym +3%, SPY +1% (rs=+2) | +1pt bull | no bonus |
| Sym -2%, SPY +1% (rs=-3) | no bonus | +1pt bear |
| Sym -1%, SPY -3% (rs=+2) | +1pt bull | no bonus |

## Choppiness gate (v7.0 I6 CHANGED)

- v6.10: active **only on 1H+** (15m/30m skipped)
- v7.0: active **on 30m+** (15m still skipped)
- 30m chop catches consolidation false breakouts that pullback-resumption triggers
- CHOP threshold (`chopIndexThresh`, default 61.8) unchanged

15m retention: pullback-resumption legitimately fires in 15m consolidation. CHOP would false-block.
